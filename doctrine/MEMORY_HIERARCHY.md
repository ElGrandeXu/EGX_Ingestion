# MEMORY_HIERARCHY

## Layers

- N1 = full proof: source URL, PDF, raw notes, command output, local clone or benchmark evidence.
- N2 = verifiable synthesis: decision note with citations, risks, tests and traceability.
- N3 = compact operational knowledge: one card that routes the next agent decision.

## Read Order

Canonical order: `N3 -> N2 -> N1`.

Start with `memory/INDEX.n3`. Read N2 only when N3 cannot support the decision. Read N1 only when proof can change the decision.

## Escalate To N1 Only For

- new ingestion;
- source audit;
- license or security review;
- test or benchmark need;
- contradiction;
- explicit user request.

## Stay In N3 When

- the card already gives verdict, risk, confidence, proof pointers and next action;
- the task is routing, triage or handoff;
- extra detail would not change the decision.
